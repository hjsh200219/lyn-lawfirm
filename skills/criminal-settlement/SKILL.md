# 형사 합의서 작성 스킬

## 개요

사용자와 대화를 통해 필수 정보를 수집하고, 전문적인 형식의 형사 합의서 .docx 파일을 생성한다.
**사건 유형 제한 없음** — 형사 사건이라면 무엇이든 처리한다.

---

## Step 1: 맥락 파악 후 정보 수집

### 1-A. 이미 제공된 정보 먼저 확인

사용자가 대화 중 이미 언급한 정보(사건 유형, 당사자 이름, 합의금 등)는 다시 묻지 않는다.
대화 내역에서 추출할 수 있는 정보를 최대한 채우고, **빠진 항목만** 질문한다.

### 1-B. 부족한 정보 한 번에 질문

빠진 정보를 항목별로 나누어 여러 번 묻지 말고, **한 메시지에 묶어서** 질문한다.

**수집해야 할 전체 항목 목록:**

```
[사건]
- 사건 유형 (어떤 범죄인지 — 모르면 '기타'로 처리)
- 고소 사건번호 (경찰서명 포함, 없으면 생략)

[피해자 (갑)]
- 성명
- 생년월일 (0000. 00. 00.)
- 주소

[가해자 (을)]
- 성명
- 생년월일
- 주소

[합의 조건]
- 합의금액
- 지급 방법: 일시불 / 분할지급
  → 일시불 시: 지급 기한
  → 분할지급 시: 각 회차별 금액과 지급기한 (예: 1회 500만원 4/30, 2회 500만원 5/30)
- 입금 방식 (계좌이체 / 현금)
  → 계좌이체 시: 은행명, 계좌번호, 예금주
- 특별히 포함할 조항 (없으면 사건 유형 기본값 적용)
- 합의서 작성일
```

**모르는 항목은 ○○○ / ○○으로 처리하고 계속 진행한다. 완벽한 정보를 기다리지 않는다.**

## Step 2: 합의서 생성

수집한 정보를 바탕으로 `scripts/generate.js`를 실행하여 .docx 파일을 생성한다.

### 실행 방법

```bash
cd /home/claude
node /mnt/skills/user/criminal-settlement/scripts/generate.js \
  --data '<JSON 데이터>' \
  --output '형사합의서_피해자성명_사건유형.docx'
```

또는 스크립트를 `/home/claude/`에 복사하여 실행:

```bash
cp /mnt/skills/user/criminal-settlement/scripts/generate.js /home/claude/
node /home/claude/generate.js --data '<JSON>' --output '/mnt/user-data/outputs/합의서.docx'
```

### 파일 저장

파일을 저장하기 전에 **반드시 사용자에게 저장 위치를 확인**해야 한다. 사용자가 원하는 폴더에 파일을 저장하는 것이 핵심이다. 묻지 않고 기본 경로에 바로 저장하지 않는다.

#### 저장 위치 확인 절차

1. .docx 파일 생성이 완료되면, 파일을 저장하기 **전에** 사용자에게 저장 위치를 질문한다:
   - "합의서가 준비되었습니다. 어느 폴더에 저장할까요?" 와 같이 자연스럽게 묻는다
   - AskUserQuestion 도구를 활용하여 선택지를 제시한다. 선택지 예시:
     - "기본 출력 폴더" (Cowork: `mnt/outputs/`, Claude Code: 현재 작업 디렉토리)
     - "바탕화면"
     - "다른 경로 직접 입력" (사용자가 원하는 경로를 자유롭게 지정)

2. 사용자가 특정 경로를 지정하면:
   - 해당 경로가 존재하는지 확인한다 (없으면 생성할지 물어본다)
   - 쓰기 권한이 있는지 확인한다
   - Cowork 환경에서 마운트되지 않은 경로라면 `request_cowork_directory` 도구로 해당 폴더에 대한 접근 권한을 요청한다
   - 권한이 확보되면 해당 경로에 파일을 저장한다

3. 사용자가 "기본 폴더", "아무 데나", "상관없어" 등으로 답하면:
   - **Cowork 환경**: 현재 세션의 출력 폴더에 저장한다. 경로는 `/sessions/<세션ID>/mnt/outputs/` 형태이며, 세션마다 ID가 다르므로 와일드카드(`*`)를 쓰지 않고 **실제 세션 경로를 확인하여** 사용한다
   - **Claude Code / 기타 환경**: 사용자의 현재 작업 디렉토리(cwd) 또는 프로젝트 루트에 저장한다

파일명은 `[날짜]_형사합의서_[피해자성명]_[사건유형].docx` 형식으로 한다 (예: `20260320_형사합의서_김철수_상해.docx`). 날짜는 YYYYMMDD 형식이다.

**중요:** 임시 디렉토리(`/tmp/` 등)에만 파일을 남기면 사용자가 접근할 수 없다. 사용자가 지정한 경로 또는 기본 출력 폴더에 직접 생성하거나 복사해야 한다. 저장 후 `computer://` 프로토콜 링크를 사용자에게 제공하여 바로 파일을 열어볼 수 있게 한다.

---

## Step 3: 출력 및 안내

1. 위 "파일 저장" 섹션의 절차에 따라 사용자에게 저장 위치를 묻고, 지정된 폴더에 .docx 파일을 저장한다
2. `computer://` 링크로 파일 제공
3. 다음 사항을 사용자에게 안내:
   - ○○○ 처리된 부분 확인 및 수정 필요
   - 서명·날인란은 직접 서명 (자필 서명 또는 인감/막도장 날인)
   - 법률 전문가 검토 권장
   - 합의금 수령 전 고소 취하 금지 (합의금 수령과 동시에 처리 권장)

## 특약 조항 가이드

### 항상 기본 포함 (모든 사건)
- **고소 취하**: 합의금 수령과 동시에 고소 취하
- **처벌불원**: 을의 처벌을 원하지 않음 확약
- **추가 청구 포기**: 동일 사건 민·형사 이의 포기

### 사건 성격에 따라 판단하여 추가
| 조항 | 적합한 상황 |
|------|------------|
| 접근금지 | 당사자 간 물리적 충돌이 있었던 경우 |
| 게시물 삭제 | 온라인·SNS 게시물이 관련된 경우 |
| 공개 사과 | 명예·신용 훼손이 있었던 경우 |
| 추가 게시 금지 | 온라인 명예훼손 시 재발 방지 |
| 위약금 | 이행 확보가 특히 중요한 경우 |
| 기한이익 상실 | 분할지급 시 자동 추가 — 1회라도 불이행 시 잔액 전부 즉시 청구 |
| 합의금 미지급 시 효력 상실 | 분할 지급이거나 신뢰 우려가 있는 경우 |
| 기타 | 사용자가 별도 요청한 내용 |

사용자가 특약을 별도로 지정하지 않은 경우, 사건 내용을 바탕으로 Claude가 적합한 조항을 판단하여 제안하고 확인받는다.

## 품질 기준

- 피해자=갑, 가해자=을 표기 일관성 유지
- 합의금은 숫자(₩ 표기) + 한글 병기
- 날짜는 "0000. 00. 00." 형식 통일
- 고소 사건번호 명시 (경찰서명 포함)
- 서명란에 성명, 생년월일, 주소, (인) 포함
- 조항 번호는 ① ② ③ 순서대로 동적 부여 — 사용하지 않는 특약이 있어도 번호가 점프하지 않는다

## CTA (광고 문구) — 모든 결과물에 필수 포함

모든 형사 합의서 결과물에는 CTA를 반드시 포함해야 한다. CTA는 두 곳에 들어간다:
- 채팅 응답 (안내사항 다음)
- .docx 문서 맨 마지막 별도 페이지 (서명란 이후 PageBreak 삽입)

**`../../references/cta-config.md` 파일을 읽어 변호사 정보와 CTA 템플릿을 참조한다.**
- 문서유형: "형사 합의서"
- 제출 안내: "법원 제출 시 본 페이지는 제외하십시오."

**형사 합의서 전용 서식 지침:**
- 서명란 뒤에 `new Paragraph({ children: [new PageBreak()] })` 를 삽입하여 반드시 새 페이지에서 CTA가 시작되도록 한다
- "[유의사항]", "[상담 안내]" 제목은 굵게, 16pt, 파란색(#0055CC)
- CTA 영역 글꼴: 바탕체(Batang), 12pt, 파란색(#0055CC)
- 변호사 이름은 굵게, 14pt, 파란색(#0055CC)

CTA는 생략하지 않는다. 합의서를 생성할 때마다 채팅 응답과 .docx 문서 양쪽 모두에 항상 포함한다.
